关于Next主题的一些配置问题

一直想要建立一个自己的博客网站，在这个周末正式实施了，通过两天的的部署大体的内容已经部署完毕，但在部署过程中遇到了一些问题，为此记录下给遇到相同问题的人一些参考。

关于next的主要配置问题可以查阅next文档来解决，这里写几个在文档中没有涉及到的问题。

1.菜单栏图标问题

​ 在官方文档中，菜单栏图标的配置是在字段__menu_icons__中的，但是在next版本的后续更新中，__menu_icons__字段仅仅用来控制是否允许显示图标，而对图标的选择则是在__menu__字段中。下方是在__主题配置文件__中对图标进行配置的实例。

注意的是在每个标签后面不能够留有空格，不然在菜单栏点击该栏目后链接中会出现一个%20导致前入指定页面发生错误。

2.侧边栏个人信息不显示

​ 这个问题比较奇葩，相信遇到的人一定不多，毕竟我搜遍全网也没有找到解决的办法，最后还是通过扒源码才解决的。在官方文档中，关于配置__作者昵称__和__站点描述__说的是只需要编辑__站点配置文件__设置其中的author与description便能够正常显示。但是我发现我配置完成并不能正常显示，在一系列操作无果后就去查看layout源码，由此发现了一些问题来解决。

首先找到侧边栏布局文件的位置，该位置在

\blog\themes\next\layout_macro

找到其中的sidebar.swig便是侧边栏的布局文件，在此前，我发现在网站的底部我的用户名可以正常显示

站点介绍也是如此，只需要遇上修改后便能够正常显示。

3.关于评论系统

​ 一开始我使用的评论系统是韩国的__必力__，但是由于墙的缘故，挂了vpn还需要加载好久，因此我换成了LeanCloud，发现可以正常使用，配置的详细步骤如下:

​ 我们的评论系统其实是放在Leancloud上的，因此首先需要去注册一个账号 点我注册

1、 注册完以后需要创建一个应用，名字可以随便起，然后 进入应用->设置->应用key ,获取你的 appid 和 appkey

2、 打开主题配置文件 搜索 valine，填入appid 和 appkey

2、在Leancloud -> 设置 -> 安全中心 -> Web 安全域名 把你的域名加进去之后重启服务便能够正常显示。
